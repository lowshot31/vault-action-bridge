# セキュリティポリシー

Vault Action Bridgeは、選択されたノートコンテンツの読み取り、設定されたモデルプロバイダーの呼び出し、ユーザーのVault内のファイルへのレビュー済み変更の適用が可能なObsidianデスクトッププラグインです。

## サポートバージョン

最新のGitHubリリースのみがセキュリティ修正の対象です。

## 脆弱性の報告

個人データを公開しない問題については、GitHub Issuesを通じて報告してください。機密性の高い詳細を含む問題については、公開の再現手順を投稿する前にメンテナーに非公開で連絡してください。

報告時に含める事項：

- プラグインバージョン
- Obsidianバージョン
- オペレーティングシステム
- 使用したプロバイダーモード（`openai-oauth`、APIキープロバイダー、またはウェブビューのみ）
- 再現手順
- 予期される動作
- 実際の動作
- Vaultファイルや認証情報が公開された可能性の有無

## セキュリティモデル

このプラグインはAIの書き込みを明示的なレビューステップの後に配置します：

1. モデルがVaultアクションJSONを提案できます。
2. プラグインがリクエストされたアクションとVault相対パスを検証します。
3. ユーザーがアクションの要約とJSONをレビューします。
4. プラグインは確認後にのみ変更を適用します。

絶対パスと`..`パストラバーサルは拒否されます。

## ネットワーク使用

ネットワーク動作はユーザー設定に依存します：

- 現在のノートと選択テキストの質問は、設定されたプロバイダーに送信されます。
- `openai-oauth`モードは、設定されたローカルプロキシURL（通常`http://127.0.0.1:10531/v1`）にリクエストを送信します。
- APIキープロバイダーモードは、選択したプロバイダーエンドポイントにリクエストを送信します。

## 認証情報の処理

APIキーとプラグイン設定は、Obsidianの`loadData()`および`saveData()` APIを使用してObsidianプラグインデータに保存されます。選択したプロバイダーが受信することを許可されていないノートや認証情報とともにこのプラグインを使用しないでください。

## ツールインストール

プラグインにはNode.js、Codex、`openai-oauth`のセットアップボタンが含まれています。これらのボタンは、ユーザーが押した後に表示されるターミナルでコマンドを実行します。プラグインはプログラムを自動的にインストールしたり、バックグラウンドで認証を行ってはなりません。

## テレメトリ

Vault Action Bridgeにはクライアント側のテレメトリやアナリティクス機能は含まれていません。
